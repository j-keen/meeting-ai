## 1. Executive Summary

**Meeting AI**는 웹 기반 실시간 AI 회의 보조 도구로, 브라우저만으로 음성 인식(STT), AI 실시간 분석, 대화형 AI 채팅, 회의록 관리를 제공하는 올인원 회의 생산성 솔루션이다.

### 핵심 투자 포인트

| 항목 | 내용 |
|------|------|
| **제품 단계** | Working Prototype (MVP+) |
| **기술 스택** | 순수 웹 기술 (Vanilla JS) + Gemini API + Web Speech API |
| **핵심 차별화** | 100% 클라이언트 사이드 처리, 설치 불필요, 프라이버시 우선 |
| **타겟 시장** | AI 회의 보조 도구 시장 (글로벌 $3.5B, 2026) |
| **비즈니스 모델** | Freemium SaaS (B2C → B2B 확장) |
| **투자 제안** | Pre-Seed $300K ~ Seed $1M |
| **투자 매력도** | ★★★☆☆ (3.5/5) - 기술 차별화 있으나 시장 진입 리스크 존재 |

### 핵심 강점
- **제로 인프라 아키텍처**: 서버 없이 운영 가능, 극히 낮은 초기 운영 비용
- **프라이버시 우선**: 모든 데이터가 사용자 브라우저(localStorage)에 저장
- **다국어 지원**: 한국어/영어 완전 이중 언어 지원 (UI + AI 분석)
- **풍부한 기능셋**: 실시간 STT, AI 분석, 채팅, 회의 준비 마법사, 오타 교정, Slack 연동

### 핵심 리스크
- Gemini API 단일 의존 (vendor lock-in)
- Web Speech API의 정확도 한계
- localStorage 5MB 제한 (확장성)
- 서버리스 구조의 B2B 전환 어려움

### 4.2 기술 혁신 포인트

1. **스마트 토큰 전략**: 이전 분석 요약 + 최근 N분 전사를 조합하여 장시간 회의에서도 비용 효율적 분석
2. **점진적 분석**: 주기적 자동 분석으로 회의 중 실시간 인사이트 제공
3. **AI 오타 교정**: Gemini API를 활용한 STT 오류 자동 학습/교정 (typo dictionary)
4. **Function Calling 기반 채팅**: AI가 맥락 추가, 메모 생성, 재분석을 직접 수행
5. **회의 준비 마법사**: 대화형 플로우로 회의 유형/안건/참석자/프롬프트 설정

## 8. Unit Economics

### 8.1 핵심 지표 (기본 시나리오 기준, Year 2-3 안정화)

| 지표 | 값 | 벤치마크 |
|------|-----|----------|
| **CAC (고객 획득 비용)** | $15-25 | SaaS 평균 $50-100 |
| **ARPU (월)** | $10-12 | 경쟁사 $8-20 범위 내 |
| **LTV (생애 가치)** | $200-400 | 평균 유지 20-40개월 기준 |
| **LTV/CAC 비율** | 8-16x | 3x 이상이면 양호 |
| **Payback Period** | 1.5-2.5개월 | 12개월 이하면 우수 |
| **월 이탈률** | 5% | SaaS 평균 5-7% |
| **Gross Margin** | 85-90% | 서버리스 구조 덕분에 높음 |
| **Net Revenue Retention** | 105-115% | 업셀 (Pro→Team) 포함 |

### 8.2 CAC 분석

Meeting AI의 PLG(Product-Led Growth) 전략에서 CAC가 매우 낮은 이유:

1. **바이럴 루프**: 회의록 공유 시 제품 노출 (Slack 전송, 이메일 공유)
2. **SEO/콘텐츠**: "AI 회의록", "회의 자동 요약" 등 검색 유입
3. **커뮤니티**: 오픈소스 또는 BYOK 모델로 개발자 커뮤니티 형성
4. **제로 온보딩 비용**: 설치 불필요, 브라우저만 있으면 즉시 사용

### 8.3 LTV 계산

```
LTV = ARPU × Gross Margin × (1 / 이탈률)
    = $10 × 0.87 × (1 / 0.05)
    = $10 × 0.87 × 20
    = $174 (보수적)

LTV (Team 업셀 포함) = $12 × 0.87 × 25 = $261 (기본)
```

## 11. 결론

### 투자 의견: **조건부 긍정 (Conditional Positive)**

Meeting AI는 기술적으로 완성도 높은 MVP를 보유하고 있으며, 프라이버시 우선 아키텍처와 한국어 특화라는 명확한 차별화 포인트를 갖추고 있다. 특히 서버 비용 없이 동작하는 제로 인프라 구조는 초기 스타트업에게 매우 유리한 비용 구조를 제공한다.

### 투자 가치 (Bull Case)
1. **프라이버시 규제 트렌드**가 Meeting AI의 클라이언트 사이드 아키텍처에 유리하게 작용
2. **한국어 AI 회의 도구 시장**에서 클로바노트의 대안으로 빠른 시장 점유 가능
3. **제로 서버 비용** 구조로 극히 높은 그로스 마진 (85-90%) 달성
4. **LLM API 비용 지속 하락**으로 AI 기능의 무료 제공 범위 확대 가능
5. **PLG 모델**로 낮은 CAC와 바이럴 성장 가능

### 투자 우려 (Bear Case)
1. **단일 API 의존** (Gemini)은 전략적 리스크
2. **B2B 전환 시 아키텍처 대규모 변경** 필요 (서버, 인증, 권한 관리)
3. **빅테크 (Microsoft Copilot, Google Meet AI)의 내장 기능** 경쟁
4. **Web Speech API의 품질 한계**로 프로 사용자 만족도 이슈 가능
5. **수익화까지의 긴 여정** (12개월 이상 예상)

### 투자 조건 (Key Conditions)
1. 3개월 내 **멀티 LLM 지원** (Gemini 의존도 탈피) 구현 계획 확인
2. 6개월 내 **서버 백엔드 + 관리형 API 키** 전환 로드맵 합의
3. **핵심 엔지니어 1명 이상** 합류 확인 또는 계획
4. **10,000 MAU** 달성 시 추가 투자 연계 (트랜치 구조 고려)
